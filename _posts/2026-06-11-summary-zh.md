## [案例研究：采用 HTML 优先与 HTMX 的网站在一夜之间用户量翻倍](https://mohkohn.co.uk/writing/html-first/) ⭐️ 8.0/10

一位开发者的案例研究详细介绍了如何采用基于 HTMX 库的 HTML 优先、渐进增强架构，使一个网站的用户量在一夜之间翻倍，并提升了性能。该方法侧重于使用标准 HTML 表单构建核心功能，并用 HTMX 增强交互性。 这个真实的成功案例为 HTML 优先和渐进增强的开发理念提供了强有力的、有数据支持的论据，挑战了复杂单页应用（SPA）框架的主导地位。它表明，更简单、更具韧性的架构可以直接带来更好的用户参与度和可访问性，可能影响更广泛的 Web 开发趋势。 所描述的架构使用标准 HTML 实现核心功能，确保网站在没有 JavaScript 的情况下也能工作，并利用 HTMX 的自定义属性来添加 AJAX 驱动的交互性作为渐进增强。社区中一个值得注意的反驳观点指向一篇名为《为单页应用辩护》的文章，凸显了现代 SPA 与经典服务器端方法之间的持续争论。

hackernews · edent · 6月10日 12:45 · [社区讨论](https://news.ycombinator.com/item?id=48475483)

**背景**: HTMX 是一个前端 JavaScript 库，它通过属性扩展 HTML，允许开发者直接在标记中访问 AJAX、WebSockets 和服务器发送事件，倡导一种超媒体驱动的方法。渐进增强是一种 Web 设计原则，它从一个稳固、普遍可访问的 HTML 基础开始，然后在功能强大的浏览器中叠加 CSS 和 JavaScript 以提供增强体验。这与单页应用（SPA）形成对比，后者通常严重依赖客户端 JavaScript 来渲染内容。
