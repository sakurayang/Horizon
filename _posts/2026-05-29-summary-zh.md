## [AI Agent 驱动服务器 CPU 市场 2030 年将超 2000 亿美元，ARM 架构快速崛起](https://pdf.dfcfw.com/pdf/H3_AP202605291822987579_1.pdf?1780043689000.pdf) ⭐️ 8.0/10

国信证券的一份研究报告预测，全球服务器 CPU 市场规模将从 2025 年的 312 亿美元增长至 2030 年的 2070 亿美元，年复合增长率达 46%。市场格局正从 Intel 和 AMD 主导的 x86 双雄争霸，转向 x86 阵营守成、ARM 阵营快速崛起的三足鼎立局面，预计 ARM 份额将从 2025 年的 11%提升至 2030 年的 27%。 这一预测揭示了计算基础设施的根本性转变，AI Agent 正使 CPU 从配套配角转变为核心主力，全面重构需求、价格和产业格局。ARM 架构凭借其低功耗、高能效及定制化优势，受到英伟达、谷歌等云厂商青睐，其崛起正在挑战长期由 x86 主导的市场格局，标志着异构计算新时代的到来。 报告预测，AI 服务器中总体的 CPU 与 GPU 配比将从 2026 年的 1:3 上升至 2030 年的 0.93:1，接近 1:1 均衡，英伟达、谷歌等巨头已在其最新平台采用 1:2 的配比。市场呈现明确场景分化：柜内 AI 训练/推理以 ARM 为主流，而柜外扩容场景则依托 x86 生态的兼容性，由 Intel 和 AMD 主导。

**背景**: 服务器 CPU 是驱动数据中心和云服务的核心处理器。以 Intel 和 AMD 为首的 x86 架构凭借其性能和庞大的软件生态，长期主导服务器市场。与之相对，ARM 架构采用开放的 IP 授权模式，允许苹果、高通、英伟达等公司设计高度定制化、高能效的芯片，这推动了其在移动端乃至服务器市场的增长。CPU 与 GPU 的配比是 AI 服务器中的关键指标，反映了为优化性能，管理并向每个 GPU 加速器输送数据所需的 CPU 数量。

## [OSCAR：面向真实场景服务的 2-bit KV Cache 量化新方法](https://www.infoq.cn/article/B36ZgoaReVDs3l05yw0z) ⭐️ 8.0/10

研究人员提出了 OSCAR（离线谱协方差感知旋转）这一新的 2-bit KV Cache 量化方法，旨在为长上下文大语言模型服务保持注意力质量。该方法已集成到 SGLang 服务框架中，在 Qwen3-4B-Thinking 模型上，相比之前的先进方法 TurboQuant，性能提升了 40.1%。 这很重要，因为 KV Cache 是长上下文大语言模型服务中内存和带宽的主要瓶颈。OSCAR 能够将 KV Cache 压缩到接近 2-bit 同时保持模型质量，可以显著降低服务成本，并支持更高效地部署具有长上下文的大模型。 OSCAR 实现了约 2.28 bits 的每 KV 元素有效比特率，并使用一个注意力感知的旋转矩阵（R = U · Hadamard · bit-reversal）将量化噪声推向对注意力分数影响较小的方向。其在 SGLang 中的实际实现采用三段式 token 池（BF16 sink、INT2 history、BF16 recent）和融合的 Triton 内核，用于在线部署。

rss · InfoQ 推荐 · 5月29日 01:00

**背景**: 在 Transformer 推理过程中，键值（KV）缓存用于存储先前 token 的中间状态以避免重复计算，但它会消耗大量 GPU 内存和带宽，对于长上下文尤其如此。像 TurboQuant 这样的量化技术可以压缩这些向量，但可能会在复杂任务上降低模型性能。SGLang 是一个为低延迟大语言模型推理设计的高性能服务框架。

文章指出，随着字节跳动“豆包手机”等产品的出现，AI 大模型提供商与传统智能手机制造商之间开始出现战略分歧。豆包手机将 AI Agent 深度集成到系统层，能跨应用执行任务，这迫使整个行业思考一个核心问题：未来的 AI 手机究竟该由谁来主导。 这很重要，因为它预示着下一代智能手机的核心价值主张和商业模式可能出现权力争夺。如果字节跳动等模型提供商成功掌控了深度集成的 AI 体验，传统硬件制造商可能被边缘化，从而重塑移动生态的利润中心和竞争格局。 豆包手机销量仅约 3 万台，在中国庞大的手机市场中微不足道，但其意义在于展示了一套完全集成、系统级的 GUI Agent 解决方案。文章指出，目前的“AI 手机”大多将 AI 功能局限于独立应用，而豆包手机的原型展示了 AI 作为真正的系统级智能体运作的潜力。

rss · 综合报道 | 极客公园 · 5月28日 15:07

**背景**: “AI 手机”通常指深度集成人工智能能力的智能手机，常通过端侧或云端的大语言模型来实现高级助手或自动化任务等功能。模型提供商（开发 AI 软件）与硬件制造商（生产设备）之间的商业关系至关重要，它决定了用户体验、数据和收入的控制权。这一概念已被炒作多年，但大多数实现仍将 AI 作为独立应用，而非操作系统的核心。
