## [NPM v12 将默认禁用包脚本以增强安全性。](https://github.blog/changelog/2026-06-09-upcoming-breaking-changes-for-npm-v12/) ⭐️ 8.0/10

即将发布的 npm v12 将引入一项重大变更，其 `allowScripts` 配置将默认设置为 `off`，从而阻止包安装脚本自动运行。这一变化遵循了 npm 11.15.0+ 中引入的相关标志（如 `--allow-scripts-pending`）以及使用 `npm approve-scripts` 来审查和允许受信任脚本的新工作流程。 这对 JavaScript 生态系统来说是一项重大的安全改进，因为包生命周期脚本中的恶意代码一直是一个常见的攻击媒介。通过要求对脚本进行显式批准，npm v12 将有助于保护数百万开发者和项目，使其免受利用安装期间自动执行脚本的供应链攻击。 这项变更允许进行细粒度控制，因为脚本可以通过允许列表按包进行批准，而不是一个全局的开关。开发者现在可以升级到 npm 11.16.0+ 来测试新行为，通过警告审查待处理的脚本，并在升级到 v12 之前使用 `npm approve-scripts` 选择性地允许它们。

hackernews · plasma · 6月9日 21:01 · [社区讨论](https://news.ycombinator.com/item?id=48467705)

**背景**: npm（Node Package Manager）是 JavaScript 运行时 Node.js 的默认包管理器，用于安装和管理依赖项。在 `package.json` 文件中定义的包脚本（如 `postinstall`）可以在安装期间自动执行命令，这功能强大，但一直被攻击者利用来运行恶意代码。其他包管理器（如 pnpm）已经通过默认禁用脚本的方式实施了类似的安全措施。

## [FCC 提议新规，要求电信运营商收集所有移动服务客户的身份信息，旨在终结匿名'一次性'手机。](https://www.404media.co/fcc-wants-to-kill-burner-phones-by-forcing-telecoms-to-get-all-customers-ids/) ⭐️ 8.0/10

美国联邦通信委员会（FCC）正在提议一项新规，要求所有电信服务提供商为移动服务收集并验证客户身份信息。此举将有效终结购买和使用匿名预付费手机（俗称'一次性'或' burner '手机）的能力。 这项提议对隐私、数字权利和执法具有重大影响，因为它将消除记者、活动人士和弱势群体用于匿名通信的一个关键工具。这标志着美国电信政策的一次重大转变，使其与金融领域常见的'了解你的客户'（KYC）法规看齐，而许多其他国家早已对移动服务实施了类似规定。 该规定目前处于提案阶段，FCC 正在征求公众意见，社区讨论中已提供了提交评论的链接。讨论中提出的一个关键反驳点是，通过使用来自不要求身份验证国家的、支持漫游的 SIM 卡，可以轻易绕过此类规定。

hackernews · berlianta · 6月9日 15:21 · [社区讨论](https://news.ycombinator.com/item?id=48462308)

**背景**: '一次性'手机（burner phone）通常指用现金购买、不与用户真实身份关联的预付费手机，能提供一定程度的匿名性。FCC 是美国负责监管州际和国际通信的政府机构。'了解你的客户'（KYC）是金融领域以及日益增多的电信领域中的标准流程，用于验证客户身份并评估潜在风险。
