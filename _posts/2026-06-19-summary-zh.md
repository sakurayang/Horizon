## [安全研究员发现一万个 GitHub 仓库分发木马恶意软件。](https://orchidfiles.com/github-repositories-distributing-malware/) ⭐️ 8.0/10

一名安全研究员发现了大约一万个分发木马恶意软件的 GitHub 仓库。这些仓库由不同的贡献者创建，名称各异，但共享一个共同模式，从而允许进行自动化检测。 这一发现突显了一个主要针对自动化 AI 智能体和依赖项搜索系统而非直接针对人类用户的重要且不断演化的安全威胁。它代表了一种复杂的软件供应链攻击，可能大规模危害自动化开发工作流和 AI 驱动的工具。 攻击策略包括创建新仓库并频繁更新（例如，每隔几小时删除并推送新提交），以便在自动化智能体的搜索结果中显得“新鲜”。该恶意软件旨在利用自动化系统在获取依赖项时对代码仓库的信任。

hackernews · theorchid · 6月18日 11:45 · [社区讨论](https://news.ycombinator.com/item?id=48583928)

**背景**: 软件供应链攻击涉及破坏软件依赖项（如库或软件包），以将恶意代码注入下游应用程序。GitHub 是托管开源代码的核心平台，因此成为主要攻击目标。自动化 AI 智能体是能够执行任务（如搜索和集成代码依赖项）且几乎无需人工监督的程序，这创造了一个新的攻击面。OWASP 社区已将保护智能体 AI 安全确定为 2025-2026 年的首要任务。

## [Transformer 奠基人之一 Noam Shazeer 离开谷歌，将于 2026 年加入 OpenAI](https://twitter.com/NoamShazeer/status/2067400851438932297) ⭐️ 8.0/10

2017 年开创性论文《Attention Is All You Need》的合著者、谷歌 Gemini 项目的联合负责人 Noam Shazeer 宣布，他将于 2026 年离开谷歌并加入 OpenAI。此举发生在他于 2024 年联合创立 Character.AI 后重返谷歌不久。 这是两家领先人工智能实验室之间一次重要的人才流动，涉及现代大语言模型基础——Transformer 架构的关键设计者之一。他的加入可能会影响 OpenAI 未来模型的战略方向和技术能力，同时也意味着谷歌 Gemini 开发工作的一次显著人才流失。 Shazeer 的过渡期定于 2026 年，表明这是一次有计划的人事变动而非立即离职。他此前于 2021 年离开谷歌并联合创立了 Character.AI，后于 2024 年通过一项据称价值 27 亿美元的许可和人才协议随该公司部分研究人员一同重返谷歌。

hackernews · lukasgross · 6月18日 00:26 · [社区讨论](https://news.ycombinator.com/item?id=48578913)

**背景**: 2017 年的论文《Attention Is All You Need》提出了 Transformer 架构，这是一种完全基于注意力机制的神经网络设计，现已成为几乎所有现代大语言模型（LLM）的基础，包括 OpenAI 的 GPT 系列和谷歌的 Gemini。相较于之前的循环神经网络（RNN）架构，Transformer 的关键创新在于能够在训练过程中实现更高效的并行处理，这对于扩大模型规模和能力至关重要。

## [谷歌开源 Agent Substrate 和 AX，旨在为 AI Agent 打造下一个 Kubernetes](https://www.infoq.cn/article/jNsfjJuAJjDzGYS51jHC) ⭐️ 8.0/10

谷歌开源了两个早期 Apache 2.0 项目：Agent Substrate，一个构建在 Kubernetes 之上、用于管理 AI Agent 生命周期的控制平面；以及 Agent eXecutor（AX），一个用于协调 Agentic 循环的分布式运行时。这些项目旨在处理数百万次亚秒级工具调用，并将突发性 Agent 工作负载的硬件效率提升高达 97%。 此举意义重大，因为它解决了规模化部署 AI Agent 的一个关键瓶颈，类似于 Kubernetes 当年解决容器编排问题，有望为蓬勃发展的 Agent 生态建立一个基础架构层。通过将 Agent 逻辑与计算资源解耦，并实现快速状态恢复，它可以为运行大规模、交互式 Agent 系统的企业大幅降低成本和提升效率。 这些项目处于非常早期的阶段（v0.0.0 和 v0.1.0），不保证向后兼容性，并且专门针对 Agent 的高频、突发性交互模式而设计，这与传统的长期运行服务不同。Agent Substrate 通过对非活跃 Agent 进行快照和挂起来实现效率提升，允许多个逻辑会话共享物理资源，恢复时间通常在几百毫秒内，而非完整的冷启动。

rss · InfoQ 推荐 · 6月18日 09:27

**背景**: AI Agent 是利用大语言模型（LLM）通过决策和使用工具（如 API 调用）来执行任务的自主程序。Kubernetes 是一个用于自动化部署、扩展和管理容器化应用程序的开源系统，已成为云原生基础设施的事实标准。大规模编排许多交互式 AI Agent 带来了新的挑战，因为它们会产生突发性、高频的工作负载，并且在等待外部输入时存在不活跃期，这是传统的、单独的 Kubernetes 等基础设施无法高效处理的。
