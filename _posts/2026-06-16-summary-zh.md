## [LinkedIn 招聘邀约暗藏后门，恶意 GitHub 仓库可执行任意代码](https://roman.pt/posts/linkedin-backdoor/) ⭐️ 8.0/10

## [Iroh 1.0 发布：支持应用直接连接的 P2P 网络库](https://www.iroh.computer/blog/v1) ⭐️ 8.0/10

Iroh 项目发布了其点对点网络库的 1.0 版本，该库使应用程序能够使用加密密钥而非 IP 地址直接连接。这个主要版本标志着稳定的 API 和核心功能集，其定位是'应用层的 Tailscale'。 这很重要，因为它通过抽象掉复杂的网络配置和 NAT 穿透，简化了去中心化应用的构建，让开发者能专注于应用逻辑。它代表了向基于身份的网络转变，对于动态的点对点服务而言，这比传统的基于 IP 的寻址更具弹性和用户友好性。 Iroh 使用 Rust 编写，目前开箱即用地支持 IPv4、IPv6 和中继传输，并具有可扩展的架构以支持自定义传输。开发者提到的一个关键限制是它尚未原生支持 WebRTC、BLE 或 LoRa，但可以实现自定义传输。

hackernews · chadfowler · 6月15日 15:13 · [社区讨论](https://news.ycombinator.com/item?id=48542480)

**背景**: 点对点网络允许设备在没有中央服务器的情况下直接通信，但它经常面临网络地址转换和防火墙穿透的挑战。Tailscale 是一个流行的解决方案，它使用 WireGuard 隧道创建安全的私有网络，但它主要在网络层运行。Iroh 旨在提供类似的连接便利性，但工作在应用层，使用加密密钥作为端点的持久身份标识。

## [开发者分享成功案例：用本地大语言模型替代云端 AI 编程助手。](https://news.ycombinator.com/item?id=48542100) ⭐️ 8.0/10

一场 Hacker News 讨论显示，多名开发者已成功将付费的云端 AI 编程助手（如 Claude 和 GPT）替换为本地运行的大语言模型，并将其用于主要开发工作。用户分享了具体配置，涉及 Qwen3.6-35B 和 Gemma 等模型，运行在从高端 Mac 到双 GPU PC 的硬件上，速度可达每秒约 150 个 token。 这一转变突显了一个日益增长的趋势：开发者正转向私有化、高性价比且自给自足的 AI 开发工作流，以减少对昂贵云 API 的依赖并解决数据隐私问题。这表明，对于许多编程任务，开源本地模型的性能和能力已成为前沿云端模型的实用替代方案。 流行的配置包括在 VS Code 中使用'Continue'扩展，搭配 Ollama 或 LM Studio 等本地后端，以及使用 GGUF 等特定模型格式以实现高效的 CPU/GPU 推理。尽管用户报告这些本地模型'不如'顶级云端模型聪明，但对于大多数日常编程任务已足够，其质量可与 8-12 个月前的云端模型相媲美。

hackernews · cloudking · 6月15日 14:46

**背景**: 本地大语言模型（LLM）是完全在用户自己的硬件上运行的 AI 模型，与 OpenAI 的 GPT 或 Anthropic 的 Claude 等云端服务相对。Ollama、LM Studio 和 llama.cpp 等工具简化了下载和运行这些开源模型的过程。'每秒生成的 token 数'（tok/s）是衡量 LLM 性能的关键指标，它反映了模型生成文本的速度，直接影响编程助手的响应速度。
