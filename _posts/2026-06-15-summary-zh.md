## [里约热内卢宣称的'自研'大语言模型实为现有模型的加权合并](https://github.com/nex-agi/Nex-N2/issues/4) ⭐️ 8.0/10

对里约热内卢市政府发布的 Rio-3.5-Open-397B 模型的分析表明，该模型并非其宣称的基于 Qwen 的自研微调模型，而是一个简单的加权合并模型，由约 60% 的 Nex-N2 Pro 和约 40% 的 Qwen3.5-397B-A17B 组成。该模型的发布未披露这一合并过程，引发了关于透明度的争论。 这一事件凸显了开源 AI 生态系统中透明度和正确归属的关键问题，模型合并正成为一种流行但往往不透明的技术。它引发了关于何为'自研'开发，以及公共机构和开发者准确说明其工作来源的道德义务的讨论。 技术分析表明，Rio 模型中的所有权重张量，在所有网络层中，都以极高的统计置信度是两种源模型的 0.6/0.4 线性混合。据报道，合并后的模型在基准测试中表现良好，证明了简单的权重插值无需额外训练（如蒸馏）即可有效。

hackernews · unrvl22 · 6月14日 15:37 · [社区讨论](https://news.ycombinator.com/item?id=48528371)

**背景**: 模型合并是机器学习中的一种技术，通过加权平均等方式，将多个预训练模型的参数（权重）组合起来，创建一个新模型，而无需在原始数据上进行昂贵的重新训练。这是一种有效结合不同模型优势的方法。在大语言模型（LLM）领域，mergekit 等工具使这一过程更加便捷。AI 模型的开源许可证（如 MIT 或 Apache 2.0）通常要求在分发衍生作品时进行正确的归属说明。

## [2014 年的演讲预言了 JavaScript 的演变与最终被取代](https://www.destroyallsoftware.com/talks/the-birth-and-death-of-javascript) ⭐️ 8.0/10

在 2014 年题为《JavaScript 的诞生与死亡》的演讲中，Gary Bernhardt 预言 JavaScript 将演变为其他语言的通用编译目标，并最终被取代。这一预言随着后来 WebAssembly（Wasm）成为主要编译目标，以及通过 Electron 等框架将 Web 技术扩展到桌面应用，被证明具有惊人的准确性。 这次演讲之所以重要，是因为它正确预见了 Web 开发的一次重大架构转变，即 JavaScript 的角色从脚本语言扩展为基础平台。其见解有助于我们理解 Web 标准的持续演变、WebAssembly 等技术背后的逻辑，以及 Web 内外软件开发的长期轨迹。 该演讲特别提到了 asm.js，这是一个早期用作编译目标的 JavaScript 低级子集，后来在很大程度上被更高效、更标准化的 WebAssembly 所取代。当前讨论中的一个关键局限是，WebAssembly 仍然缺乏直接的 DOM 操作能力，通常需要 JavaScript 作为“粘合”代码来实现完整的 Web 互操作性。

hackernews · subset · 6月14日 12:38 · [社区讨论](https://news.ycombinator.com/item?id=48526661)

**背景**: JavaScript 是一种高级编程语言，主要用于在 Web 浏览器中创建交互效果。“编译目标”是指编译器将源代码翻译成的格式或语言；例如，许多语言可以编译成 JavaScript 或 WebAssembly。WebAssembly（Wasm）是一种二进制指令格式，被设计为编程语言的可移植编译目标，使高性能应用程序能够在 Web 上运行。“通用编译目标”的概念是指一种单一的、低级的格式，许多不同的高级语言都可以编译到该格式，从而促进跨生态系统的互操作性和代码复用。
